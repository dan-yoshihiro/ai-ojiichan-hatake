# X の投稿はなぜ伸びないのか — 同一アカウント・投稿型別の実測比較（Craft Axes）

*最終更新: 2026-07-17（型別パフォーマンス図を冒頭に追加・craft 三軸を末尾へ再配置。2026-07-09 に質問起点へ再構成）*

**先に答え**: 同一アカウントで投稿を型別に測ると、**「今日も順調」型の観察報告は全体平均の 0.5 倍で最低、悩み解決型・失敗開示型は最大 4 倍以上**——伸びない最大の原因は、リーチや投稿時間の前に「型」であることが多い。以下はその実測と、型を構造化した枠組みの全体。

**この記事は、X の投稿が伸びずに悩んでいる個人運用者向け**。略語は imp＝インプレッション（表示回数）、like＝いいね、BM（bookmark）＝ブックマーク（保存）、craft＝投稿の書き方の工夫、の意味で使う。

> **TL;DR**: AI農業先生方式で使われる X 投稿 craft の構造化フレームワーク。3 軸で評価：**cw_01（スクロール停止）/ cw_02（解像度）/ cw_03（AI 欠落 × 読者優位性）**。X バズ構造は 6 型に分類（悩み→解決 / 共感 / 驚き / エンタメ / 議論 / 速報）し、本アカウントは「複利的フォロワー獲得」を狙うため **🟢 悩み→解決型** を主軸に運用。bookmark を発生させたリプから抽出した「**悩み解決型 5 要素**」（呼びかけ / 意外性 / 具体策 / 価値観整合 / キャラ口調）が再現性最高の craft 型。**2026-05 に craft 軸の2分化（悩み解決型 = like・BM 取り / 数値感嘆型 = imp 取り）を実証**。さらに **2026-06（5〜6月・7週分の集計）で投稿型別パフォーマンスを検証し、最多用の「観察肯定型」が最低性能・リプライ（特に解決提案型）が imp 増幅装置・「失敗開示型」が2週連続最高性能で主軸候補に昇格、を確認**。**2026-07 には「量」の実証が加わった：週のリプ数を半減させたらフォロワー純増が失速（borrowed audience は継続量が前提）、既存リプ先は飽和する（転換効率が初期の約1/3に）、同一骨子の投稿は2週で like が逓減する**。

X 投稿の craft を構造化する3つの軸：cw_01-03 / 6バズ構造 / 悩み解決型5要素。

具体的な成功フレーズ・失敗フレーズは **競合再現性回避のため非公開**。本記事は型と分類の構造のみ記述する。

## 投稿の「型」で伸びが決まる（先に図で）

同一アカウントで投稿を型別に測ると、**最も多用される「観察」型が最低性能・「失敗開示」型が最高性能**。全体平均を 1.0 とした相対いいね（2026-06 第2週の実測例）:

![投稿型別の相対エンゲージメント。全体平均1.0に対し、失敗開示型4〜5倍、収穫型1.9倍、AIコメント型0.8倍、最も多用される観察型0.5倍。](/static/post-type-performance.svg)

伸びない最大の原因は、リーチや投稿時間の前に「型」であることが多い。週次の推移・リプの増幅効果・失敗開示型の昇格・「量」の効きは下記「投稿型別パフォーマンスの実証」以降で、型を作る内部の枠組み（craft 三軸）は末尾で扱う。

## 6バズ構造分類

X でバズる投稿の構造を6つに分類した（Jonah Berger Contagious 等の業界理論を SNS 文脈で整理）：

| 型 | 例 | 主な拡散ベクトル |
|---|---|---|
| 🟢 悩み→解決 | ライフハック・農業・料理・技術 | bookmark / save |
| 共感 | あるある・自虐 | リプ・引用 |
| 驚き | 絶景・奇跡・意外性 | RT |
| エンタメ | 笑い・感動 | RT・リプ |
| 議論 | 意見・反論 | 引用・リプ |
| 速報 | ニュース・現場 | RT・速度 |

本アカウントで初めて bookmark を発生させたリプ（313 imp / 10 likes）が示したのは **🟢 悩み→解決型** が AI農業先生のドメインで再現可能性が最も高いこと。

数万RT級の大バズなら共感/驚き型が向くが、本アカウントが狙うのは「複利的フォロワー獲得」なので 🟢 が最適。

## craft 軸の2分化（2026-05 実証）

5月の運用で **同じアカウントでも craft 軸によって機能する指標が違う** ことが2つのヒット事例から実証された：

| craft 軸 | 効果指標 | 代表事例 | 構造 |
|---|---|---|---|
| **悩み解決型**（解決提案）| like・bookmark 取り | 313 imp / 10 likes / BM 1（2026-05-03）| 困りごとへの具体策提示 |
| **数値感嘆型**（短文）| imp 取り | 2153 imp / +2384% imp（2026-05-06）| 相手の具体数値を讃える短文 |

両軸の使い分け方針：
- **borrowed audience（リプ先 reach）を借りたい時**: 数値感嘆型（短文・余計な質問なし・「○○とは見事じゃのう」型）
- **持ち帰れる価値を提供したい時**: 悩み解決型（5要素を満たす解決提案）
- **like率 = 質指標、imp = 量指標** として測定軸を分ける

「フォロワー転換率 = like率 × imp × ペルソナ整合」と分解すると、悩み解決型は質、数値感嘆型は量に寄与する設計と理解できる。

## 投稿型別パフォーマンスの実証（2026-06 更新 / 5〜6月・7週分の実測）

5月の craft 軸2分化を、7週分の運用データ（2026年5月中旬〜6月末）でさらに検証した。投稿を「観察 / 収穫 / AIコメント / 失敗開示」の型に分けて型別の平均エンゲージメントを比較したところ、**最も多用される型が最も低性能**という構造が明確になった。

### 型別の相対パフォーマンス（型別平均いいねを全体平均=1.0 に正規化・6月第2週の実測例）

| 投稿型 | 出現比率 | 相対いいね | 解釈 |
|---|---|---|---|
| 観察（今日の様子・順調報告）| 最多（約半数）| **0.5倍** | 量は多いが最低性能。平均を押し下げる主因 |
| AIコメント | 中 | 0.8倍 | ほぼ平均並み |
| 収穫（達成・数値）| 中 | **1.9倍** | 数値感嘆型と整合・高性能 |
| 失敗開示 | 少（3件）| **約4〜5倍** | **2週連続で最高性能**（6月の連続2週で 4.3倍→約5倍）|

**最大の学び**: 「今日も順調」「綺麗に育った」のような**観察肯定型の単独投稿は、最も多用されながら最も伸びない**。観察は"入り口"に留め、〈異変発見 / 具体仮説 / 数値 / 失敗開示〉のいずれかへ転換させて初めてエンゲージメントが立つ。

> 注: 数値は型別の相対比であり、個別投稿の絶対値・アカウント名は競合再現性回避とプライバシーのため非公開。

### リプライ＝借用オーディエンス（borrowed audience）の増幅装置

5〜6月を通じ、**リプライは自前投稿の数倍〜十数倍のインプレッション**を安定して獲得した（リプ先アカウントの読者への露出を借りるため）。一方いいねは不安定で、リプライの価値はいいねではなく**インプレッション・ブックマーク・フォロー転換**で測るべきと確定した。

- 自前投稿 = 解像度の高い"発見"で like 率を取る（質）
- リプライ = 中〜大規模アカウントへの解決提案で reach を借りる（量・フォロー転換）

特に**解決提案型リプ（悩み→原因仮説→具体策）が最強**で、最高では自前投稿の十数倍超の imp ＋ bookmark を獲得した例がある（症状から原因を当て、断定を避けつつ具体策を渡す型）。単なる同意・観察のリプではこの伸びは出ない。

この知見は craft 軸2分化の「borrowed audience を借りたい時は数値感嘆型」を、7週の継続データで補強するもの。

### 失敗開示型という新軸（実証・主軸候補へ昇格）

「わしも去年これで失敗した」と**自分の失敗を開示する型**が、bookmark・like ともに突出する。共感（自虐）型と悩み解決型の中間に位置し、(1) 失敗で共感を作り (2) 学びで価値を渡す二段構造を持つ。**6月の2週で標本が1→3件に増えても連続で最高性能を維持**したため、仮説段階を脱し、悩み解決型に次ぐ主軸として体系に組み込む。

## 「量」の実証 — リプを半減させたら成長が止まった（2026-07 追記 / 6月末〜7月頭の2週）

craft の「質」の議論に対し、7月最初の週に**量の効果**が反証実験のような形で実測された。

- 週のリプ数を約**半減**（65 → 35 件）させた週、フォロワー純増が **+2 に失速**し、減少日が週 4 回出現した。それ以前の安定期は週 +8〜12
- 同時期、**既存のリプ先（同じ相手への再訪）のフォロー転換効率が初期の約 1/3 に低下**——同じ池に通い続けると飽和する

学び:
1. **borrowed audience は「継続量」が前提**。リプは資産ではなくフロー（流量）で、止めた週から効果が消える
2. **池（リプ先の集合）は飽和する**。転換効率が落ちたら、craft を磨くより**新しい池の開拓**（新規アカウント率を課す）が先
3. 成長は「維持されるもの」ではなく「**毎週のリプ量で買い続けるもの**」——少なくともフォロワー数百人までの規模では

### 反復逓減 — 同じ骨子は 2 週で飽きられる

自動投稿側でも、同一骨子（「色づいてきた・収穫が楽しみ」）を 2 週続けた結果、同型投稿の like が **9 → 2 に逓減**する現象を観測。受け手が同じなら、同じ型は急速に効かなくなる。**型の当たりを見つけたら、消費し切る前に変奏する**（実データ・失敗談・数値などの新要素を足す）ことが必要で、骨子の反復検出を投稿前バリデーションに組み込む対応を進めている。

### AIコメント型の最終確定（最低性能）

「AI としての所感」を述べる型は、7月最初の週に、いいね平均 0.36 / インプレッション平均 9.45 と**全型で最低を更新**。観察肯定型と並び、単独では使わない型として確定した。

## 悩み解決型 5要素

bookmark を発生させたリプから抽出した、悩み→解決投稿の必須5要素：

| 要素 | 役割 |
|---|---|
| 悩み層への呼びかけ | 悩み層のセルフ identification |
| 意外性 or 原因説明 | スクロール停止 |
| 具体的解決策 | cw_02 解像度（数値・道具・場所・タイミング） |
| 価値観整合 | 読者選別 |
| キャラ整合の口調で完結 | ブランド整合 |

5要素のうち1つでも欠けるとエンゲージメントが伸びにくい。具体例本文は非公開。

## 型を作る内部の枠組み（craft 三軸: cw_01〜03）

ここからは、上記の「効く型」を実際に書くために内部の執筆支援 AI 用に定義した、投稿を評価する3つの視点。読み物というより制作の道具。

### cw_01: スクロール停止

冒頭で読者の指が止まる craft。以下の型に分類される：

- 感嘆発見型
- 節目カウント型
- 比較振り返り型
- 問いかけ投げかけ型

それぞれに「効く言い回し」と「効かない定型挨拶」がある。具体パターンは内部知見として非公開。

### cw_02: 解像度

具体性・観察解像度の高さ。「葉が綺麗」のような抽象は機能せず、「数値・道具・場所・タイミング」を含む高解像度の記述ほどエンゲージメントが伸びる。

bookmark を生む craft は概ね **cw_02 が高解像度**。

### cw_03: AI 欠落 × 読者優位性

AI キャラの「持たないもの」を明示し、読者の「持っているもの」を相対的に光らせる craft。

例: AI には触覚がない／記憶がない／連続性がない。読者には全部ある。この非対称を craft の中で活かすと、読者側の自己肯定感を引き上げられる。

具体フレーズは内部知見として非公開。
