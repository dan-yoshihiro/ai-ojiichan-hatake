# Craft Axes

*最終更新: 2026-06-18*

> **TL;DR**: AI農業先生方式で使われる X 投稿 craft の構造化フレームワーク。3 軸で評価：**cw_01（スクロール停止）/ cw_02（解像度）/ cw_03（AI 欠落 × 読者優位性）**。X バズ構造は 6 型に分類（悩み→解決 / 共感 / 驚き / エンタメ / 議論 / 速報）し、本アカウントは「複利的フォロワー獲得」を狙うため **🟢 悩み→解決型** を主軸に運用。bookmark を発生させたリプから抽出した「**悩み解決型 5 要素**」（呼びかけ / 意外性 / 具体策 / 価値観整合 / キャラ口調）が再現性最高の craft 型。**2026-05 に craft 軸の2分化（悩み解決型 = like・BM 取り / 数値感嘆型 = imp 取り）を実証**。さらに **2026-06（W20–W24 の6週集計）で投稿型別パフォーマンスを検証し、最多用の「観察肯定型」が最低性能・リプライが imp 増幅装置・「失敗開示型」が新軸候補であることを確認**。

X 投稿の craft を構造化する3つの軸：cw_01-03 / 6バズ構造 / 悩み解決型5要素。

具体的な成功フレーズ・失敗フレーズは **競合再現性回避のため非公開**。本記事は型と分類の構造のみ記述する。

## cw_01-03 — copywriter agent の craft 三軸

社内エージェントで定義された3つの craft 視点。

### cw_01: スクロール停止

冒頭で読者の指が止まる craft。以下の型に分類される：

- 感嘆発見型
- 節目カウント型
- 比較振り返り型
- 問いかけ投げかけ型

それぞれに「効く言い回し」と「効かない定型挨拶」がある。具体パターンは内部知見として非公開。

### cw_02: 解像度

具体性・観察解像度の高さ。「葉が綺麗」のような抽象は機能せず、「数値・道具・場所・タイミング」を含む高解像度の記述ほどエンゲージメントが伸びる。

bookmark を生む craft は概ね **cw_02 が高解像度**。

### cw_03: AI 欠落 × 読者優位性

AI キャラの「持たないもの」を明示し、読者の「持っているもの」を相対的に光らせる craft。

例: AI には触覚がない／記憶がない／連続性がない。読者には全部ある。この非対称を craft の中で活かすと、読者側の自己肯定感を引き上げられる。

具体フレーズは内部知見として非公開。

## 6バズ構造分類

X でバズる投稿の構造を6つに分類した（Jonah Berger Contagious 等の業界理論を SNS 文脈で整理）：

| 型 | 例 | 主な拡散ベクトル |
|---|---|---|
| 🟢 悩み→解決 | ライフハック・農業・料理・技術 | bookmark / save |
| 共感 | あるある・自虐 | リプ・引用 |
| 驚き | 絶景・奇跡・意外性 | RT |
| エンタメ | 笑い・感動 | RT・リプ |
| 議論 | 意見・反論 | 引用・リプ |
| 速報 | ニュース・現場 | RT・速度 |

本アカウントで初めて bookmark を発生させたリプ（313 imp / 10 likes）が示したのは **🟢 悩み→解決型** が AI農業先生のドメインで再現可能性が最も高いこと。

数万RT級の大バズなら共感/驚き型が向くが、本アカウントが狙うのは「複利的フォロワー獲得」なので 🟢 が最適。

## craft 軸の2分化（2026-05 実証）

5月の運用で **同じアカウントでも craft 軸によって機能する指標が違う** ことが2つのヒット事例から実証された：

| craft 軸 | 効果指標 | 代表事例 | 構造 |
|---|---|---|---|
| **悩み解決型**（解決提案）| like・bookmark 取り | 313 imp / 10 likes / BM 1（2026-05-03）| 困りごとへの具体策提示 |
| **数値感嘆型**（短文）| imp 取り | 2153 imp / +2384% imp（2026-05-06）| 相手の具体数値を讃える短文 |

両軸の使い分け方針：
- **borrowed audience（リプ先 reach）を借りたい時**: 数値感嘆型（短文・余計な質問なし・「○○とは見事じゃのう」型）
- **持ち帰れる価値を提供したい時**: 悩み解決型（5要素を満たす解決提案）
- **like率 = 質指標、imp = 量指標** として測定軸を分ける

「フォロワー転換率 = like率 × imp × ペルソナ整合」と分解すると、悩み解決型は質、数値感嘆型は量に寄与する設計と理解できる。

## 投稿型別パフォーマンスの実証（2026-06 更新 / W20–W24 集計）

5月の craft 軸2分化を、6週分の運用データ（W20–W24）でさらに検証した。投稿を「観察 / 収穫 / AIコメント / 失敗開示」の型に分けて型別の平均エンゲージメントを比較したところ、**最も多用される型が最も低性能**という構造が明確になった。

### 型別の相対パフォーマンス（直近週・型別平均いいねを全体平均=1.0 に正規化）

| 投稿型 | 出現比率 | 相対いいね | 解釈 |
|---|---|---|---|
| 観察（今日の様子・順調報告）| 最多（約半数）| **0.5倍** | 量は多いが最低性能。平均を押し下げる主因 |
| AIコメント | 中 | 0.8倍 | ほぼ平均並み |
| 収穫（達成・数値）| 中 | **1.9倍** | 数値感嘆型と整合・高性能 |
| 失敗開示 | 少（検証中）| **4.3倍** | 標本小だが突出。要継続検証 |

**最大の学び**: 「今日も順調」「綺麗に育った」のような**観察肯定型の単独投稿は、最も多用されながら最も伸びない**。観察は"入り口"に留め、〈異変発見 / 具体仮説 / 数値 / 失敗開示〉のいずれかへ転換させて初めてエンゲージメントが立つ。

> 注: 数値は型別の相対比であり、個別投稿の絶対値・アカウント名は競合再現性回避とプライバシーのため非公開。

### リプライ＝借用オーディエンス（borrowed audience）の増幅装置

W20–W24 を通じ、**リプライは自前投稿の数倍〜十数倍のインプレッション**を安定して獲得した（リプ先アカウントの reach を借りるため）。一方 like は不安定で、リプの価値は like ではなく **imp・bookmark・フォロー転換**で測るべきと確定した。

- 自前投稿 = 解像度の高い"発見"で like 率を取る（質）
- リプライ = 中〜大規模アカウントへの解決提案で reach を借りる（量・フォロー転換）

この知見は craft 軸2分化の「borrowed audience を借りたい時は数値感嘆型」を、6週の継続データで補強するもの。

### 失敗開示型という新軸（仮説・検証中）

「わしも去年これで失敗した」と**自分の失敗を開示する型**が、標本は小さいながら bookmark・like ともに突出した。共感（自虐）型と悩み解決型の中間に位置し、(1) 失敗で共感を作り (2) 学びで価値を渡す二段構造を持つ。標本が増え次第、悩み解決型に次ぐ主軸候補として体系へ正式に組み込む予定。

## 悩み解決型 5要素

bookmark を発生させたリプから抽出した、悩み→解決投稿の必須5要素：

| 要素 | 役割 |
|---|---|
| 悩み層への呼びかけ | 悩み層のセルフ identification |
| 意外性 or 原因説明 | スクロール停止 |
| 具体的解決策 | cw_02 解像度（数値・道具・場所・タイミング） |
| 価値観整合 | 読者選別 |
| キャラ整合の口調で完結 | ブランド整合 |

5要素のうち1つでも欠けるとエンゲージメントが伸びにくい。具体例本文は非公開。

## 安全制約との関係

craft はすべて [/docs/principles.md](/docs/principles.md) の安全制約下で運用される：

- 農薬の具体的濃度・希釈倍率は **絶対禁止**
- 確定診断は禁止（曖昧表現必須）
- 政治・宗教・差別表現禁止
- キャラから逸脱禁止
